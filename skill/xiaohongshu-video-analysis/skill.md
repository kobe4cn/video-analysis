# 小红书单视频分析技能

此技能用于深度分析单个小红书视频的内容、数据和质量,生成完整的分析报告。

## 使用说明

**输入**:小红书视频笔记URL，或者要求分析某个小红书博主的作品，如果不是直接输入的笔记URL，那么要先使用浏览器获取笔记链接
**输出**:完整的markdown格式视频分析报告

使用方法:
- 提供小红书视频链接或者指明要分析小红书的视频
- 示例:"分析这个视频 https://www.xiaohongshu.com/explore/xxxxxxxx"，"分析小红书博主科技少女小妖最新的作品"

## 分析流程

### 阶段1:初始化和任务规划

### 阶段2:打开视频页面并提取数据

#### 2.1 打开视频页面

```bash
工具: mcp__mcp-router__new_page
参数:
  - url: {视频的完整笔记URL(含xsec_token)}
  - timeout: 30000
```

#### 2.2 等待页面加载

```bash
工具: mcp__mcp-router__wait_for
参数:
  - text: 点赞
  - timeout: 10000
```

#### 2.3 获取页面快照

```bash
工具: mcp__mcp-router__take_snapshot
参数: {}
```

#### 2.4 提取视频基础数据

从快照中提取以下数据:

**必须提取的数据**:
1. 视频标题
2. 发布时间(注意格式转换)
3. 点赞数(注意"1.1万" = 11000)
4. 收藏数
5. 评论数
6. 分享数(如果有)
7. 视频时长(如果有显示)
8. 博主昵称
9. 笔记URL(当前页面URL)

**数据格式转换规则**:
- "1.1万" → 11000
- "29万" → 290000
- "557" → 557
- "昨天12:25" → 计算为具体日期
- "11-02" → 补充为"2024-11-02"
- "3天前" → 计算为具体日期

**保存数据到变量**:
```
视频数据 = {
  标题: xxx,
  发布时间: xxx,
  点赞: xxx,
  收藏: xxx,
  评论: xxx,
  分享: xxx,
  时长: xxx,
  博主: xxx,
  笔记URL: xxx,
  视频MP4_URL: 待获取,
  AI分析结果: 待分析
}
```

#### 2.5 计算互动率

```
互动率 = (收藏数 + 评论数) / 点赞数 × 100%
```

### 阶段3:提取视频文件URL

#### 3.1 调用视频URL提取技能

```bash
工具: Skill
参数:
  - skill: xiaohongshu-extract-video-url
  - args: {视频的笔记URL}
```

#### 3.2 提取MP4文件URL

**从技能返回结果中提取MP4 URL**
- 返回格式通常包含video_url或mp4_url字段
- 保留完整的URL(包含时效性参数)

**保存URL到变量**:
```
视频数据.视频MP4_URL = {提取到的MP4 URL}
```

#### 3.3 验证URL有效性

```bash
# 简单验证URL是否可访问
工具: Bash
参数:
  command: curl -I "{MP4_URL}" 2>&1 | head -1
```

如果URL返回404或其他错误,记录错误但继续后续流程。


### 阶段4:AI视频内容分析

#### 4.1 使用AI视频分析工具analyze_video

```bash
工具: mcp__zai-mcp-server__analyze_video
参数:
  - video_source: {视频的MP4 URL}
  - prompt: "请详细分析这个小红书视频并提供以下信息:

1. 视频内容主题:判断视频的核心主题和内容类型(教程/测评/分享/剧情等)
2. 视觉呈现效果:分析画面质量、视觉效果、剪辑节奏、画面布局、色彩运用
3. 内容质量评估:评估内容的原创性、实用性、信息密度(高/中/低)
4. 讲解风格:分析视频的表达方式、讲述风格(专业/亲民/幽默/情感化等)
5. 完整口播内容:尽可能完整地提取视频中的所有口播、旁白、对话和讲解内容
6. 核心观点:提炼视频的3-5个核心观点和知识点
7. 目标受众:判断视频的主要受众群体(年龄、兴趣、需求等)
8. 视频时长:准确记录视频的实际时长
9. 亮点总结:总结视频最吸引人的1-3个亮点
10. 改进建议:提出2-3条具体的改进建议(内容/制作/运营层面)"
```

#### 4.2 保存AI分析结果

**将AI分析结果保存到变量**:
```

  主题: xxx,
  视觉效果: xxx,

    原创性: xxx,
    实用性: xxx,

  },
  讲解风格: xxx,
  口播内容: xxx,
  核心观点: [xxx, xxx, xxx],
  目标受众: xxx,
  时长: xxx,
  亮点: [xxx, xxx],
  改进建议: [xxx, xxx]
}
```

**错误处理**:
- 如果AI分析失败,记录错误信息:"AI分析失败: {错误原因}"
- 继续生成报告,在报告中说明AI分析部分缺失


### 阶段5:生成完整分析报告

#### 5.1 评估数据表现

**数据表现评级标准**:
- **优秀**: 互动率 > 10% 或 点赞数 > 10000
- **良好**: 互动率 5-10% 或 点赞数 5000-10000
- **一般**: 互动率 2-5% 或 点赞数 1000-5000
- **待提升**: 互动率 < 2% 或 点赞数 < 1000

## 错误处理和恢复机制

### 视频页面打开失败(404错误)

**错误**: 访问小红书笔记时返回404错误
1. 确认网络连接是否正常
2. 尝试从博主主页点击该笔记,获取新的完整链接
3. 如果仍然失败,向用户报告错误并说明需要重新获取链接

### 视频页面打开失败(其他错误)

**错误**: 无法打开视频详情页(非404错误)
**处理**:
1. 检查URL格式是否正确
2. 确认网络连接是否正常
3. 尝试重新打开页面
4. 如果仍然失败,向用户报告错误

### URL提取失败

**错误**: xiaohongshu-extract-video-url技能失败
**处理**:
1. 记录错误: "视频URL提取失败: {原因}"
2. 标记视频MP4_URL为"提取失败"
3. 在报告中说明无法进行AI视频内容分析
4. 继续生成基于基础数据的报告

### AI分析失败

**错误**: mcp__zai-mcp-server__analyze_video工具失败
**处理**:
1. 记录错误: "AI分析失败: {原因}"
2. 标记AI分析结果为"分析失败"
3. 在报告中说明AI分析部分缺失
4. 基于已提取的数据生成简化版报告

### 数据提取不完整

**错误**: 某些数据字段无法提取
**处理**:
1. 记录缺失字段
2. 在报告中标注"数据缺失"
3. 基于可用数据继续分析
4. 在摘要中说明数据完整性

## 重要约束

1. **禁止下载视频文件**
2. **必须提取完整的基础数据**(点赞、收藏、评论、分享)
3. **必须使用xiaohongshu-extract-video-url技能获取视频URL**
4. **必须进行AI视频内容分析 video_analysis**(除非URL提取失败)
5. **报告必须包含完整的优化建议**(至少8条,涵盖4个层面)
6. **数据格式必须准确转换**(万 → 乘以10000)
7. **报告必须保存为markdown格式**
8. **遇到错误要记录并继续,不能中断流程**
